## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

üìß Contact: **security@yourdomain.com**  
(If you don't have a domain yet, use: yourgithubusername@gmail.com)

### What to include in the report

Please provide:

- Description of the vulnerability
- Steps to reproduce
- Proof of concept (if available)
- Impact assessment (what an attacker could do)
- Suggested fix (optional)

### Response timeline

- Initial response: **within 48 hours**
- Status update: **within 5 days**
- Fix (if confirmed): **as soon as possible depending on severity**

### Disclosure policy

- Please **do not publicly disclose** the vulnerability until it is fixed.
- Responsible disclosure will be acknowledged.
- Valid reports may be credited in the project.

### Severity handling

We follow a risk-based approach inspired by **OWASP / CVSS**:

- Critical ‚Üí Immediate fix
- High ‚Üí Priority patch
- Medium ‚Üí Scheduled fix
- Low ‚Üí Best-effort
